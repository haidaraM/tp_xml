Etape 3 : Mise en forme du corpus Le Petit Prince [XSLT]

Vous allez maintenant produire une feuille de style XSL qui permet la transformation de l'instance du document XML en document HTML. Cette feuille va comprendre des éléments de mise en forme HTML et des commandes XSLT permettant de gérer le contenu. Le fichier HTML généré devra être le plus similaire possible à ce résultat.

Utilisez sous XMLPad la fonction File/New/XSL Stylesheet pour créer le fichier feuille de style, sauvegardez-le ensuite sous le nom de petit_prince_3IF.xsl dans le répertoire de votre projet.

Voici les caractéristiques attendues de cette feuille de style (recopiez l'en-tête et les premières règles puis inspirez-vous en pour créer les règles suivantes) :

Si vous voulez tester votre feuille de style dans un navigateur, rajoutez dans l'en-tête de votre document XML un appel à la feuille de style que vous allez créer

<?xml-stylesheet type="text/xsl" href="PetitPrince.xsl"?>

3.1 En-tête du fichier

L'en-tête de la feuille de style a été créé automatiquement par le système.

<?xml version = "1.0" encoding = "UTF-8"?>
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

Si vous voulez que le résultat s'affiche en HTML, ajouter la ligne :

<xsl:output method="html" />

comme premier élément fils de l'élément <xsl:stylesheet>. Pour afficher le résultat en XML, remplacer 'html' par 'xml' (notamment pour les premières questions).

3.2 Contexte HTML du document

On décrit un modèle général qui regroupe les autres. Ce modèle s'applique à la racine (défini par le chemin XPath : " / "). Il débute par les informations HTML qui seront reportées dans le document résultat. On retrouve ainsi la structure d'un document HTML classique. Les modèles suivants seront appliqués à l'emplacement de la commande <xsl:apply-templates/>

<xsl:template match="/">
<html>
<head>
<title>
<xsl:value-of select="//titre"/>
<xsl:text> de </xsl:text>
<xsl:value-of select="//auteur"/>
</title>
</head>
<body style="background-color:white;">
<xsl:apply-templates/>
</body>
</html>
</xsl:template>

3.3 Affichage du titre en alignement centré entre des balises html <h1> et en bleu

Le premier modèle va définir le traitement à apporter à l'élément "titre". Il s'agit ici des instructions Html : alignement, taille et couleur de la police.

<xsl:template match="titre">
<h1 style="text-align:center; color:blue;">
<xsl:apply-templates/>
</h1>
</xsl:template>

3.4 Affichage de l’auteur de l'ouvrage <xsl:template match="auteur">

De la même manière, vous allez décrire la mise en page de l'élément "auteur"

  • en taille <h2>
  • centré et en italique style="text-align:center; font-style: italic;"
  • avec des sauts de ligne avant et aprés <br/>

3.5 Affichage des informations sur les traitements <xsl:template match="info_traitements">

  • entre balises <blockquote>
  • couleur de police = darkgreen
  • saut de ligne entre les données
    • But du TP du "date" : "but"
    • Auteurs : "1er auteur" et "2ème auteur"
    • Email du responsable : "email"
  • affichage d'une ligne horizontale: <hr/>

Deuxième partie : XQuery

Vous allez maintenant exploiter les données du fichier XML. L'objectif est d'obtenir à chaque fois un document XML bien formé, donc avec un seul élément racine. Les requêtes doivent être intégrées entre des balises XML, par exemple :

<phrases>{
for $p in //phrase
return $p
}</phrases>

<modalites>{
for $m in distinct-values(//phrase/@modalite)
order by $m
return <modalite>{data($m)}</modalite>
}</modalites>

BaseX est un gestionnaire de base de données XML capable d'interpréter des requêtes XQuery. En utilisant le menu File/New, créez une nouvelle base de données à partir du fichier petit_prince_3IF.xml avec les options par défaut proposées par BaseX. Affichez l'onglet de saisie de requêtes (Edit/Query Panel). Il y a deux modes de requêtes: les requêtes "simples" uniquement en XPath, et les requêtes XQuery (compatible XPath). Plus d'informations sur le site de la documentation de BaseX.org.
